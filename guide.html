Guide to Parameterized JUnit Tests

The following is a brief guide to using JUnit to create parameterized tests, and is adapted from the video content of the Udemy course, JUnit and Mockito Crash Course, by Bharath Thippireddy.


Overview of JUnit

JUnit is a Java framework used to write unit tests. In JUnit3, it is necessary to extendjunit.framework.TestCaseand follow certain naming conventions.

However, JUnit4 removes this necessity and provides annotations

  • @Test
  • @Ignore
  • @Before
  • @BeforeClass
  • @After

The@Beforemethod is called before each test method, and the@Aftermethod is called after each test method. Note that the test methods are run in any order.

The following are static methods defined in theorg.junit.Assertpackage:

  • assertNotNullcauses a test to fail if an object is null
  • assertEqualscauses a test to fail if two objects do not have the same values
  • assertSamecauses a test to fail if two objects are not references to the same object

To access these methods, invoke the following import statement:
import static org.junit.Assert*;

To run the tests, JUnit provides aBlockJUnit4ClassRunnerclass, which

  1. scans through the annotations and runs the setup and cleanup methods
  2. runs the test methods
  3. reports which tests have passed and which have failed

Tests are run from IDEs like Eclipse and Netbeans or with build tools like Ant or Maven, which internally use theBlockJUnit4ClassRunnerclass, the default JUnit test runner.


Parameterized JUnit Tests

Introduced in JUnit4, parameterized JUnit tests enable us to test different data sets in one method.

Steps to writing parameterized JUnit tests:

  1. Identify the input data and results
  2. Create, in the test class, fields for the inputs and result
  3. Create a constructor that takes in the inputs and result, and initialize the fields accordingly
  4. Create a static method that feeds the data, and annotate it with the@Parameters annotation
  5. Create the test method (which will be receiving the data sets one at a time)
  6. Annotate the test class with@RunWith(Parameterized.class)

Example: Testaddmethod ofCalculatorclass

Below is a type of JUnit test that you are likely already familiar with. It is a non-paramerized test for theaddmethod of theCalculatorclass.
That is, thetestAddmethod has aCalculatorobject calculate the sum of two hard-coded input values, and uses the staticassertEqualsmethod to test that the calculated sum matches a certain hard-coded expected value.

public class CalculatorImplTest {
    private int num1, num2, expectedResult;
    private Calculator calc = new CalculatorImpl();
    
    // Replace this test
    @Test
    public void testAdd() {
        assertEquals(30, calc.add(10, 20));
        assertEquals(60, calc.add(25, 35));
    }
}

In the above method, there are multiple assert statements. If one fails, the whole test will fail, without indicating which of the assert statements passed and which failed.
The objective is to replace this test method with a new one that does not contain hard-coded values for input and expected output.


Step 1: Identify the input data and results

Inputs: num1,num2
Result: result (should be num1+num2)

Step 2: Create fields for the inputs and result

public class CalculatorImplTest {
    private int num1, num2, expectedResult;
    ...
}

Step 3: Create a constructor that takes in the inputs and result, and initialize the fields accordingly

public class CalculatorImplTest {
    private int num1, num2, expectedResult;  
    public CalculatorImplTest(int num1, int num2, int expectedResult) {
        this.num1 = num1;
        this.num2 = num2;
        this.expectedResult = expectedResult;
    }
    ...
}

Step 4: Create a static method that feeds the data, and annotate it with the@Parameters annotation

JUnit looks for apublic staticmethod, annotated with@Parameters, that returns specifically aCollection ofArrayobjects.

When the test method containsninputs, each array should containn+1elements, with the firstnelements representing the inputs and the last element representing the expected output. 

public class CalculatorImplTest {
    ...  
    @Parameters
    public static Collection getTestData() {
        return Stream.of(
            new Integer[]{1, 2, 3},     // 1 + 2 = 3
            new Integer[]{3, 4, 7}      // 3 + 4 = 7
        ).collect(Collectors.toList());
    }
    ...
}

Step 5: Create the test method (which will be receiving the data sets one at a time)

public class CalculatorImplTest {
    ...
    Calculator calc = new CalculatorImpl();
    ... 
    @Test
    public void testAdd() {
        assertEquals(expectedResult, calc.add(num1, num2);
    }   
}

Step 6: Annotate the test class with@RunWith(Parameterized.class)

@RunWith(Parameterized.class)
public class CalculatorImplTest {
    private int num1, num2, expectedResult;
    Calculator cal = new CalculatorImpl();

    public CalculatorImplTest(int num1, int num2, int expectedResult) {
        this.num1 = num1;
        this.num2 = num2;
        this.expectedResult = expectedResult;
    }

    @Parameters
    public static Collection getTestData() {
        return Stream.of(
            new Integer[]{1, 2, 3},     // 1 + 2 = 3
            new Integer[]{4, 5, 6},     // 4 + 5 != 6
            new Integer[]{3, 4, 7}      // 3 + 4 = 7
        ).collect(Collectors.toList());
    }

    @Test
    public void testAdd() {
        assertEquals(expectedResult, calc.add(num1, num2);
    }
}

When you run your tests, JUnit knows at runtime that it needs to use theorg.junit.runners.Parameterizedclass to run the tests.

This class knows how to invoke thepublic staticmethod annotated with the@Parametersannotation (in this case thegetTestDatamethod), and to get each dataset (i.e. each Array) from the Collection.

For each dataset in theCollectionreturned by the@Parametersmethod, JUnit invokes the test class constructor (in this caseCalculatorImplTest) to create a new test class object, with the values from the dataset as the fields of the test class instance.

For example, for dataset{1, 2, 3}, JUnit creates a newCalculatorImplTestinstance whose num1,num2, andexpectedResultfields are set to 1, 2, and 3 respectively.

Output

✔ testAdd[0]
✕ testAdd[1]
✔ testAdd[2]

The output indicates which tests passed and which tests failed.

Since 1 + 2 = 3, the first test passed.
Since 4 + 5 != 6, the second test failed.
Since 3 + 4 = 7, the third test passed.
